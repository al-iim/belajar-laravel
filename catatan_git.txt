Saat kita membuat project Laravel, kita direkomendasikan untuk menggunakan Git sebagai source code management nya
Kita juga bisa melihat ada file seperti .gitignore yang otomatis dibuat di project Laravel, untuk memberi tahu, bagian mana yang tidak perlu di commit ke dalam Git repository
Jika teman-teman belum terbiasa dengan Git, saya sarankan mengikuti kelas Git saya sudah saya buat
https://kelas.programmerzamannow.com/p/git-untuk-pemula-sampai-mahir 

--- awal perlakuan ---
git init (inisiasi awal pengaktifan git)
git add . (memasukan semua folder dan file di project)
git commit -m "Commit 1" ("Commit 1" namanya terserah)
git remote add origin https://github.com/al-iim/belajar-laravel.git (buat dulu reponya lalu disesuaikan alamat repo nya)
git push -u origin master (di dorong di cabang utama)
--- perlakuan lanjutan ---
git add . (memasukan folder dan file yang telah di buat / ubah)
git commit -m "Commit 2" (Nama Commit 2,3,4 dst)
git push -u origin master (masukan di cabang utama)
